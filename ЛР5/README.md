# Лабораторная работа 5 - Логика приложения

## Описание
На данном этапе реализована логика приложения "Weather Notes". Добавлена функциональность для работы с заметками и получения данных о погоде через API.

## Что реализовано

### 1. Логика работы с заметками
- Создание новых заметок
- Редактирование существующих заметок
- Удаление заметок
- Отображение списка заметок
- Просмотр деталей заметки

### 2. Интеграция с погодным API
- Получение текущей погоды по координатам
- Отображение температуры, описания и иконки погоды
- Обработка ошибок при запросах к API

### 3. Навигация между экранами
- Переход с главного экрана к созданию/редактированию заметок
- Переход к просмотру деталей заметки
- Возврат с обновлением данных

## Структура проекта
```
lib/
├── models/
│   ├── note.dart       # Модель заметки
│   └── weather.dart    # Модель погоды
├── services/
│   ├── note_service.dart    # Сервис для работы с заметками
│   └── weather_service.dart # Сервис для получения погоды
├── screens/
│   ├── home_screen.dart      # Главный экран с заметками и погодой
│   ├── note_editor_screen.dart # Экран создания/редактирования заметки
│   └── note_detail_screen.dart # Экран просмотра заметки
└── main.dart
```

## Реализованные функции
- Получение списка заметок
- Добавление новой заметки
- Редактирование заметки
- Удаление заметки
- Получение погоды через API
- Обновление интерфейса после изменений

## Используемые технологии
- Dart
- Flutter
- HTTP-запросы к API
- Асинхронное программирование

## Запуск приложения
1. Убедитесь, что у вас установлен Flutter
2. Выполните `flutter pub get` для установки зависимостей
3. Запустите приложение с помощью `flutter run`
